DESCRIPTION

v.in.ghcn downloads station locations and climate time series from the NOAA Global Historical Climatology Network (GHCN) and imports them into GRASS GIS.

Station locations are imported as a vector point map (layer 1 attribute table). When a time series is requested, records are written to a secondary SQLite table named <output>_timeseries in the mapset database, keyed on the GRASS category (cat) and station_id.

By default, the module searches for stations within the current computational region. An explicit list of station IDs may be supplied instead via the stations option. No authentication is required; data are fetched directly from NOAA NCEI over HTTPS.

When domain= is given, the module runs a two-pass spatial expansion that guarantees the basin centroid is enclosed by stations with actual data in every temporal period of the requested date range.

NOTES

Time series behaviour

Time series download is the default behaviour. Use -l to import station locations only.

If neither start_date nor end_date is given, the full available record for each station is downloaded. If only one bound is given, the other defaults to the station record start or today, respectively.

Frequency and data sources

frequency=SourceElementsAccess
daily GHCNd PRCP, SNOW, SNWD, TMAX, TMIN, and others Per-station gzip CSV (HTTPS, no auth)
monthly GHCNm v4 precipitation PRCP only Per-station CSV (HTTPS, no auth)

Monthly temperature (TAVG) is distributed as a single 170 MB global flat file and is not supported. Use r.in.gridmet or r.in.prism for gridded monthly temperature.

Hourly data is not supported. GHCNh does not expose per-station bulk files; access requires the NCEI web API with date-bounded queries. More importantly, GHCNh contains synoptic meteorological variables (temperature, humidity, wind, pressure) and does not include precipitation. For hourly surface observations, use the NOAA Integrated Surface Database (ISD) instead.

For monthly imports, stations are pre-filtered using the GHCNd element inventory to identify those with any PRCP record. Stations present in GHCNd but absent from the GHCNm precipitation dataset are silently skipped (the two datasets have different coverage).

Units

GHCN raw values for PRCP, TMAX, and TMIN are stored in tenths of the standard unit. v.in.ghcn converts these to mm and degrees C on import. SNOW and SNWD are already in mm and are not modified.

Quality flags

With q_flags=strict (default), only records with a blank quality flag (passed all NCEI quality checks) are imported. Use q_flags=all to include records regardless of QC status.

Station filtering and spatial expansion

Stations are filtered in three stages:

  1. Bounding box (current region, expanded by padding= and any adaptive expansion) or explicit stations= list.
  2. Element presence: only stations with at least one requested element in the GHCN inventory are retained.
  3. Record length: if min_years= is set, only stations with at least that many years of overlap with the requested date range are downloaded. This filter controls what is downloaded; it does not affect spatial hull checks (see below).

If min_stations= is set, the bounding box is expanded in 0.5° steps until the requested number of stations is available for every requested element.

padding= applies a fixed initial expansion to the region bounding box before any adaptive expansion begins. It does not count against the max_distance= budget.

Domain enclosure (two-pass spatial expansion)

When domain= is given, the module guarantees that the basin centroid is spatially enclosed by stations with data in every temporal period of the requested date range. This requires scipy and runs two passes:

Pass 1 (inventory-based): Before downloading any data, the bounding box is expanded until the convex hull of stations with any inventory record for each element in the requested period encloses the basin centroid. The hull is checked independently for each period of length granularity= years (default 10) within start_date–end_date. Stations with even one day of inventory coverage in a period contribute to that period's hull, regardless of min_years=.

Pass 2 (data-based): After downloading, the module queries the SQLite time series table to verify that stations with actual records in each period enclose the centroid. If gaps remain and budget allows, the bounding box is expanded further and additional stations are downloaded. Pass 2 is the authoritative check; it reflects only downloaded stations (those passing min_years= if set).

Both passes share the max_distance= and max_iterations= budget. Expansion stops when either limit is reached. Gaps that cannot be closed (budget exhausted, or no surrounding stations exist) produce warnings but do not abort — early decades of long records often have genuinely sparse coverage regardless of bounding box size.

The granularity= parameter controls the period length used for temporal hull checks. The default of 1 year checks annually: a station must have at least some data in a given year to contribute to that year's hull. Larger values (e.g. 10) are coarser but may be appropriate for climatological studies where annual gaps are acceptable.

min_coverage= controls how much data a station must have within each period to count toward the Pass 2 hull. The value is a fraction of days in the period [0–1]; the default 0 requires only a single record. Setting min_coverage=0.1 requires at least roughly 36 records per year, which excludes stations with only occasional observations while retaining seasonal gauges: a rain gauge that does not operate in winter (~250 records/year ≈ 0.68) and a snow gauge that does not operate in summer (~150 records/year ≈ 0.41) both pass comfortably, while a station with one or two records per year (≈ 0.003–0.005) does not. min_coverage is applied only in Pass 2; Pass 1 uses the GHCN inventory (first/last year only) and cannot count daily records.

Time series table schema

cat        INTEGER   -- GRASS category (links to station geometry)
station_id TEXT      -- GHCN station identifier
datetime   TEXT      -- ISO 8601 date (YYYY-MM-DD)
element    TEXT      -- PRCP, SNOW, SNWD, TMAX, TMIN, ...
value      REAL      -- converted to standard units (mm or degrees C)
q_flag     TEXT      -- quality flag (NULL if blank/passed)
An index on (cat, element, datetime) is created automatically.

Querying the time series

# All PRCP records for cat 1
db.select sql="SELECT datetime, value FROM gauges_timeseries WHERE cat=1 AND element='PRCP'"

# Annual precipitation totals for all stations
db.select sql="SELECT station_id, substr(datetime,1,4) AS year, SUM(value) AS prcp_mm \
  FROM gauges_timeseries WHERE element='PRCP' GROUP BY station_id, year"

EXAMPLES

Import station locations and full PRCP record within the current region: 
v.in.ghcn output=gauges
Import PRCP and TMAX for 2000–2020, requiring at least 15 years of record: 
v.in.ghcn output=gauges elements=PRCP,TMAX start_date=2000-01-01 end_date=2020-12-31 min_years=15
Import specific stations by ID: 
v.in.ghcn output=msp stations=USW00014922,USC00218450 start_date=1980-01-01
Import monthly precipitation for 1950–2020: 
v.in.ghcn output=gauges_mo frequency=monthly start_date=1950-01-01 end_date=2020-12-31
Import station locations only (no time series): 
v.in.ghcn output=gauges -l
Import PRCP, TMAX, and TMIN for a watershed from 1950 to present, expanding the search until stations enclose the basin in every year. Stations with fewer than ~10% daily coverage in a year are excluded from the hull check to avoid counting stations with only occasional observations: 
v.in.ghcn output=ghcn_stations elements=PRCP,TMAX,TMIN \
    start_date=1950-01-01 min_coverage=0.1 \
    domain=my_basin
Same as above but using 10-year periods instead of annual (coarser check, fewer expansion steps): 
v.in.ghcn output=ghcn_stations elements=PRCP,TMAX,TMIN \
    start_date=1950-01-01 min_coverage=0.1 \
    granularity=10 domain=my_basin
